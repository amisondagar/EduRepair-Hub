# EduRepair-Hub

# Description

A real-world campus facility reporting and operations tracking system designed to solve a major university problem: handling broken infrastructure efficiently. This portal digitizes the entire workflow, allowing students to report issues on the spot while granting maintenance administrators full control over tracking, updating, and purging system logs.

# Features

- Smart Form Options: Dropdown categories change automatically depending on the campus zone you click (Classroom structural fixes vs Technical Lab faults).
- Admin & Student Views: Students are kept on a strict submission timeline, while Admins get a special toggle button to switch between student and admin dashboards.
- Permanent Local Memory: Full `localStorage` integration that keeps your tickets, logs, and logged-in user profiles saved even after page refresh or logout.
- Full Admin Control: Control center for administrators to update ticket statuses (Pending, In Progress, Resolved) or permanently delete old records using a "Purge Log" button.

# Tech Stack & Tools Used

- Frontend Structure: HTML5 (Semantic Architecture)
- Styling Engine: Tailwind CSS v3 (Play CDN Workflow with custom production log muting patches)
- Animation Matrix: Pure JavaScript (Vanilla DOM Engine)
- Visual Layer FX: Particles.js Web Mesh Emitter

# Example Workflows

- Stage 01 (Classroom Zone): Clicking "File Structural Ticket" automatically locks options to: `Furniture / Seating`, `Plumbing Defect`, or `Electrical Outage`.
- Stage 02 (Laboratory Zone): Clicking "File Systems Ticket" automatically shifts option selections to: `Lab Systems`, `AV Equipment`, or `Electrical HVAC`.
- Admin Panel: Changes ticket telemetry data states across the database or deletes records cleanly to free local cache storage.
